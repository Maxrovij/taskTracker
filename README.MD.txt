# Описание API для трекера задач java-sprint7
________________________________________________________________________________________________________________________

### Примечание:
 Все ответы приложения с информацией о полях задачи, будь то Эпик, Подзадача или простая задача,
 выводятся в формате JSON с использованием Dto. Dto имеет универсальный вид и содержит в себе все поля задач,
 как общие, так и специальные. Например, у Эпика есть поле, содержащее список ID всех его подзадач, а у Подзадачи есть
 поле, содержащее ID Эпика, которому она принадлежит. В случае вызова информации о простой задаче, эти поля будут
 присутствовать в теле ответа, но будут содержать в себе null, как и другие неинициализированные поля.
 Пример тела ответа с информацией о задаче:
{
	"id": "afc6f04d-3559-491d-9ac7-28aa552eaed7",
	"type": "TASK",
	"status": "NEW",
	"name": "Test Name",
	"title": "Test Title",
	"duration": null,
	"startTime": null,
	"endTime": null,
	"motherTaskId": null,
	"subTasksId": null
}
 Для формирования представления задачи на стороне клиента следует ориентироваться на поле "type" и заполнять поля,
 соответствующие типу запрошенной задачи.
 Поля "duration" и "startTime" должны содержать значения в миллисекундах. Для "startTime" будет сформировано
 представление метки времени исходя из количества переданных миллисекунд от UNIX Эпохи.
 Поле "endTime" - рассчетное и используется только при передаче информации для клиента. Прямому изменению не подлежит.

## Эндпоинты:
  Для работы с этим API предусмотрены следующие эндпоинты:

** 1 - Получение списка всех добавленных задач в порядке приоритета по времени начала **
  запрос
       "GET/tasks"
  вернет список всех добавленных задач в формате JSON. Задачи сортируются в порядке приоритета по
  времени начала. Если время начала задачи не установлено - первыми выводятся Эпики, далее все остальные задачи.
  Если задачи не были добавлены - вернется список со значением "null".

** 2 - Получение списка истории просмотров **
  Когда пользователь просматривает задачи в трекере, они добавляются в историю просмотров.
  Запрос
    "GET/history"
  вернет список задач в формате JSON. Список содержит 10 последних просмотренных задач и
  формируется в порядке вызова задач, независимо от статуса, типа, времени начала задачи и чего-либо еще.
  Если задачи не были добавлены - вернется список со значением "null".
________________________________________________________________________________________________________________________

** 3 - Операции с простыми задачами типа TASK**
 Операции с простыми задачами производятся через эндпоинт
    "/tasks/task"

* Для добавления простой задачи необходимо отправить запрос
    "POST/tasks/task"
  с телом запроса в формате JSON. В теле необходимо указать название и описание задачи. Название указывается в
  элементе name, а описание - в title. Пример тела запроса:
{
    "name":"Test name",
    "title":"test title"
}

* Для получения информации о простой задаче с добавлением в историю просмотров необходимо отправить запрос
  "GET/tasks/task/?id={айди задачи}".

* Для изменения информации о задаче (добавления времени начала, продолжительности, смене статуса, изменения названия
  и описания) необходимо отправить запрос:
    "PATCH/tasks/task/?id={айди задачи}"
  В телезапроса указываются поля, значения которых надо изменить. Пример тела запроса:
{
	"status": "IN_PROGRESS",
	"name": "NEW Name",
	"title": "NEW Title",
	"duration": 90000,
	"startTime": 1643983828
}

* Для удаления задачи необходимо отправить запрос
    "DELETE/tasks/task/?id={айди задачи}"

** 4 - Операции с Эпик задачами типа EPIC **
 Операции с Эпик задачами производятся через эндпоинт
    "/tasks/epic"

* Для добавления Эпик задачи необходимо отправить запрос
    "POST/tasks/epic"
  с телом запроса в формате JSON. В теле необходимо указать название и описание Эпик задачи. Название указывается в
  элементе name, а описание - в title. Пример тела запроса:
{
    "name":"Test name",
    "title":"test title"
}

* Для получения информации об Эпик задаче с добавлением в историю просмотров необходимо отправить запрос
  "GET/tasks/epic/?id={айди Эпик задачи}".

* Для изменения информации об Эпик задаче (изменения названия и описания) необходимо отправить запрос:
    "PATCH/tasks/epic/?id={айди Эпик задачи}"
  В телезапроса указываются поля, значения которых надо изменить. Пример тела запроса:
{
	"name": "NEW Name",
	"title": "NEW Title",
}
*Примечание! Все поля Эпика, кроме названия и описания являются рассчетными и не подлежат прямому редактированию!*
*Поля "status","duration","startTime","endTime" рассчитываются исходя из соответствующих значений полей Подзадач Эпика*

* Для удаления Эпик задачи необходимо отправить запрос
    "DELETE/tasks/epic/?id={айди Эпик задачи}"
*Примечание! При удалении Эпика будут автоматически удалены все его подзадачи!*

** 5 - Операции с Подзадачами типа SUBTASK **
Операции с SUBTASK задачами производятся через эндпоинт
    "/tasks/subtask"

* Для добавления простой задачи необходимо отправить запрос с указанием айди Эпика, к которому нужно добавить Подзадачу:
    "POST/tasks/subtask/?id={айди Эпика}"
  с телом запроса в формате JSON. В теле необходимо указать название и описание Подзадачи. Название указывается в
  элементе name, а описание - в title. Пример тела запроса:
{
    "name":"Test name",
    "title":"test title"
}

* Для получения информации о Подзадаче с добавлением в историю просмотров необходимо отправить запрос
  "GET/tasks/task/?id={айди Подзадачи}".

* Для изменения информации о Подзадаче (добавления времени начала, продолжительности, смене статуса, изменения названия
  и описания) необходимо отправить запрос:
    "PATCH/tasks/subtask/?id={айди Подзадачи}"
  В телезапроса указываются поля, значения которых надо изменить. Пример тела запроса:
{
	"status": "IN_PROGRESS",
	"name": "NEW Name",
	"title": "NEW Title",
	"duration": 90000,
	"startTime": 1643983828
}

* Для удаления задачи необходимо отправить запрос
    "DELETE/tasks/subtask/?id={айди Подзадачи}"
